    Preguntas del laboratorio 1:
  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • Respuesta: La diferencia principal entre Internet y la World Wide Web es que internet es la red de computadoras las cuales tienen conexión entre sí, mientras que la World Wide Web puede ser definido como el conjunto de páginas que están sobre la red de computadoras
  • ¿Cuáles son las partes de un URL?

    • Respuesta: las partes que conforman a una URL son el protocólo, dominio, carpeta, recurso, parametros.
  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Respuesta: El propósito de los métodos HTTP son los siguientes:

    • GET: Sirve para solicitar una representación de un recurso el cual es específico, dichas solicitudes solo son para la obtención de datos.

    • HEAD: Sirve para solicitar una representación de un recurso similar a la del método GET, sin embargo, no tiene el cuerpo de la respuesta.

    • POST: Sirve para enviar datos o una entidad al servidor creando un recurso.

    • PUT: Sirve para cambiar o alterar todas las representaciones actuales del recurso objetivo, con la carga de la solicitud.

    • PATCH: Sirve para hacer cambios parciales a un recurso.

    • DELETE: Sirve para borrar un recurso en específico.
  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • Respuesta: El método que se debe utilizar es el método post, esto se debe a que con ese método se usa para enviar datos al servidor creando un recurso, siendo más seguro, porque el mensaje se manda en el cuerpo del mensaje.
  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • Respuesta: El método que se debe utilizar es el método GET, esto se debe que al utilizar GET el cual es una petición, en el cual el recurso es solicitado por medio de la URL al servidor web.
  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Respuesta: La respuesta HTTP con código 200 significa que la solicitud ha tenido éxito, y una respuesta 200 se puede almacenar de forma predeterminada, de igual forma el significado de exito depende del metodo de solicitud http. Esto significa que no ocurrio un error.
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Respuesta: No es responsabilidad del desarrollador corregir el sitio web, debido a que el error 404 hace referencia a que una solicitud contiene un error, es decir, el servidor no ha podido encontrar nada que coincida con la URL de solicitud
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Respuesta: Si es responsabilidad del desarrollador corregir el sitio web, debido a que el error 500 hace referencia a que el servidor se encontró con una condición inesperada que le impide cumplir la solicitud

    • Respuesta: Las principales diferencias de HTML 4 y HTML 5 son: Para definir un juego de caracteres en html5 es introducido el atributo para la etiqueta meta charset=”UTF-8″, Se simplifíca el doctype, algunos atributos de html 4 son despreciados, al igual que algunos atributos como name cambian, nuevos atributos y elementos son introducidos, al igual que HTML5 puede incluir elementos de SVG y MathML.

    • Respuesta: Los componentes de estructura y estilo que tiene una tabla, son cuatro contenedores principales, el cual es table, osea el contenedor principal, thead para el encabezado, tbody donde va el cuerpo de la tabla y el tfoot para el pie de página, dentro de tbody puede ir tr el cual representa a las filas que contienen a las celdas y td las cuales representan las celdas.   • Sobre el ciclo de vida y desarrollo de los sistemas de información:

    • ¿Cuál es el ciclo de vida de los sistemas de información?
      • Respuesta:El ciclo de vida de los sistema de información puede ser definido como las fases de analisís, diseño para el desarrollo óptimo de software. Las fases que incluye el ciclo son nacimiento, desarrollo, operación, mantenimiento y muerte.

    • ¿Cuál es el ciclo de desarrollo de sistemas de información?
      • Respuesta: El ciclo de desarrollo de sistemas de información es un método que facilita la realización de sistemas de información, tiene el propósoto de que los gestores puedan planificar el proceso de diseño en base a los requisitos del usuario, otra función que tiene el poder de definir el tiempo de desarrollo e inversión en proyecto. Las fases de este ciclo son siete: Planificación del concepto, definición de requisitos, diseño, fase de desarrollo y pruebas, puesta en marcha, operaciones y mantenimientos y disposición
    Preguntas del laboratorio 3:
  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Respuesta: Mi recomendación como ingeniero de software, sobre el uso de !important en un CSS es que sea medido, controlado y limitado a las necesidades del sitio, y siempre usarse en elementos particulares para garantizar el buen funcionamiento, su uso es cuando se tiene una regla precedente a las demás, por ejemplo, cuando una clase interactua con un selector primario que es de estilo diferente.
  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Respuesta: Debido a la posible obstrucción de la visibilidad de los elementos del sitio web, tal es como un texto que no resalte del fonde
  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

    • Respuesta: La recomendación al elegir unidades depende de la necesidad, ya que % al ser una unidad relativa, funciona muy bien cuando se trabaja con proporciones de elementos o dentro de contenedores y esto mismo hace que se adapte a diferentes tamaños de pantalla adecuando la proporción, el uso de px funciona como una medida relativa pero destaca en proporcionar exactitud en la visualización de elementos de un tamaño definido, en el caso de pt o punto se usa cuando un elemento en pantalla quiere ser puesto en tamaño real, es decir no es una unidad relativa sino establece medidas fijas.
  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Respuesta: la versión minimizada del CSS mejora el rendimiento porque remueve bytes del archivo lo que hace un proceso de descarga, análisis y ejecución con más velocidad.
