Preguntas del laboratorio 6:

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Se trata de una buena práctica debido a las posibles alteraciones que podría hacer un usuario al archivo html cambiando las validaciones que son especificadas en sí mismo produciendo una vulnerabilidad.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Para poder saltar la seguridad de las validaciones hechas con un JavaScript, se tienen que desactivar los scripts en el buscador.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Se considera como una buena práctica debido a las implicaciones que tiene bloquear un script más allá de saltar validaciones, ya que cuando se bloquean los scripts igual la mayoría de las funciones de la página dejarán de funcionar, por lo que tener estos scripts activados es necesario para el correcto funcionamiento de la aplicación web. Debido a esto el validar usando JavaScript es una buena práctica porque se siguen eliminando las vulnerabilidades.
